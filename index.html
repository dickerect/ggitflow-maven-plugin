Gitflow Maven Plugin

Ggitflow-maven-plugin provides Maven support for Vincent Driessen's branching model http://nvie.com/posts/a-successful-git-branching-model/.

Think of Ggitflow-maven-plugin as the maven-release-plugin for the Gitflow branching model.

The plugin uses GGitflow, therefore it uses the git console directly under the hood rather than JGit.

Maven settings

Add the following to your Maven settings.xml

<pluginGroups>
   <pluginGroup>com.dkirrane.maven.plugins</pluginGroup>
</pluginGroups>

Sample POM configuration

Add the following to your top-level parent aggregator POM. The configuration is almost identical to the maven-jgitflow-plugin. However, under the hood the Ggitflow-maven-plugin invokes git commands directly. It does NOT use JGit. Why use JGit if a git command-line is provided for every Mac, Windows & Linux OS?

<plugin>
    <groupId>com.dkirrane.maven.plugins</groupId>
    <artifactId>ggitflow-maven-plugin</artifactId>
    <version>1.3</version>
    <configuration>
        <!-- Optional Config For all parameters see: -->
        <!-- http://dkirrane.github.io/ggitflow-maven-plugin/plugin-info.html -->
        <!--
        <prefixes>
            <masterBranch>master</masterBranch>
            <developBranch>development</developBranch>
            <featureBranchPrefix>feature/</featureBranchPrefix>
            <releaseBranchPrefix>release/</releaseBranchPrefix>
            <hotfixBranchPrefix>hotfix/</hotfixBranchPrefix>
            <supportBranchPrefix>support/</supportBranchPrefix>
            <versionTagPrefix>v-</versionTagPrefix>
        </prefixes>
        <msgPrefix>[ggitflow-maven-plugin]</msgPrefix>
        <msgSuffix>+review @reviewer123</msgSuffix>
        <enableFeatureVersions>true</enableFeatureVersions>
        <releaseProfiles>my-profile1, my-profile2</releaseProfiles>
        <pushFeatures>true</pushFeatures>
        <pushReleases>true</pushReleases>
        <pushHotfixes>true</pushHotfixes>
        <pushSupport>true</pushSupport>
        <skipBuild>true</skipBuild>
        <skipDeploy>true</skipDeploy>
        <skipTests>true</skipTests>
        -->
    </configuration>
</plugin>

NOTE, if you're using Windows, since the ggitflow-maven-plugin commands uses the git console it must be run from a git console such as git BASH http://msysgit.github.io/

Start & Finish a Feature branch

mvn ggitflow:feature-start
mvn ggitflow:feature-finish

Start & Finish a Release branch

mvn ggitflow:release-start
mvn ggitflow:release-finish

Start & Finish a Hotfix branch

mvn ggitflow:hotfix-start
mvn ggitflow:hotfix-finish

Start a Support branch

mvn ggitflow:support-start
